# 不用寫程式就能打造的 AI 分身：我用 4 步驟克隆了「寶玉」

> 當你的偶像太忙沒時間回你，你可以選擇等待，或者——直接複製一個他。

---

## 一個瘋狂但合理的想法

「我想請寶玉做我的 24 小時 AI 首席顧問。」

這是實踐哥 MinLi 的想法。寶玉 (@dotey) 是中文 AI 圈最具影響力的技術專家之一，但他的時間顯然比黃金還貴重。

於是 MinLi 做了一個很 Geek 的決定：**既然請不到真人，那就克隆一個數字版。**

24 小時候命、秒回訊息、永遠不會累，而且——最關鍵的是——**敢於對你說「不」**。

---

## 數字分身 vs 普通 AI：差別在哪裡？

你可能會說：「這不就是 RAG（檢索增強生成）嗎？把資料塞給 AI 讓它回答？」

錯。差別在於 **「靈魂」**。

| 普通 RAG | 數字分身 |
|---------|---------|
| 只會複述資料內容 | 能夠獨立判斷 |
| 回答中規中矩 | 有明確的風格和立場 |
| 永遠說「好的」 | **敢於說「這是偽需求」** |
| 知識的搬運工 | 思維模式的複製 |

MinLi 測試時問了一個產品經理最愛的問題：「我想做全自動推特運營 Agent，監控、回覆、發推、根據數據調整語氣，實現全自動 IP 運營。」

數字寶玉的回答：

> **「Stop. 這是一個典型的『偽需求』，而且極度危險。」**
>
> 聽起來很酷：全自動、閉環、數據驅動。但在工程實踐上，這屬於「為了自動化而自動化」的過度設計，而且極大概率會變成一個生產電子垃圾的機器人。

MinLi 說：「這一刻我確信，它比 99% 的產品經理都要清醒。」

這就是數字分身的價值——**不是複製知識，而是複製思維方式、決策模式、工程審美，還有那份敢於挑戰的勇氣。**

## 4 步驟打造你的 AI 分身

MinLi 公開了他的完整 Workflow，我們來拆解這個「平民版」的數字分身製造流程：

### Step 1：數據抓取（回旋鏢攻擊）

**目標：** 收集目標人物的所有公開輸出

MinLi 的作法：
- 用 **baoyu-skills**（寶玉自己開發的工具）抓取過去 3 個月的技術推文和長文
- 下載 GitHub 的 README 和源碼
- 用 Codex 寫腳本自動化整個流程

**關鍵洞察：** 用對方自己的武器來收集資料，既準確又完整。

### Step 2：RAG 架構（暴力美學）

這一步是整個流程的亮點。

**傳統 RAG 的做法：**
```
清洗數據 → 文本切片 → 計算 Embedding → 向量檢索
   ↓           ↓            ↓              ↓
 累          煩           貴            複雜
```

**MinLi 的新做法：**
```
資料 → 直接塞進 NotebookLM → 開始對話
```

對，就是這麼簡單。

幾百條推文？整理成 4 個文件丟進去。  
GitHub repo？貼上連結讓 NotebookLM 自己處理。

**NotebookLM 在這裡扮演了「RAG 懶人包」的角色：**
- ✅ 自動處理 PDF、Markdown、網頁
- ✅ 自動切片和索引
- ✅ 自然語言對話（不用學向量檢索語法）
- ✅ 自動標註來源

目前支援最多 300 個文件，對個人知識庫來說綽綽有餘。

### Step 3：注入靈魂（最關鍵的一步）

這是技術與藝術的交界。

普通 AI 只會說客套話，因為它沒有「立場」。要讓數字分身像真人，必須提取那個人的「思維向量」。

MinLi 的做法：
1. 在 NotebookLM 中反覆對話，觀察寶玉的回覆模式
2. 提取關鍵特徵：「反教條主義」、「工程導向」、「敢於質疑」
3. 在 Gemini 中創建 **Gem**（Google 的客製化 AI 功能）
4. 連接 NotebookLM 作為知識後端
5. 寫入精心設計的 System Instructions

這些 Instructions 不是簡單的「你是一個專家」，而是包含了：
- 價值觀（什麼是對的，什麼是錯的）
- 決策框架（如何分析問題）
- 溝通風格（怎麼說話，什麼語氣）
- 邊界條件（什麼情況下會拒絕）

### Step 4：圖靈測試

完成後，必須進行測試。不是問「你知道什麼」，而是問「你怎麼想」。

MinLi 用了那個「全自動推特 Agent」的問題來測試——這是一個典型的「聽起來很酷但實際很爛」的想法。

數字寶玉的反應證明它不只是資料庫，而是**真正理解了寶玉的工程哲學**。

## 這對你有什麼用？

你可能會想：「我又不是追星，我要數字分身幹嘛？」

讓我們換個角度思考：

### 場景 1：知識管理升級

你過去寫過幾十篇文章、幾百條推文、錄過幾十小時的影片。這些都是你的「數字遺產」，但現在它們分散在各處，連你自己都找不到。

用這個 Workflow：
- 把所有內容餵給 NotebookLM
- 之後直接問：「我上個月寫過什麼關於 AI Agent 的觀點？」
- 再也不用翻 10 個資料夾找一篇舊文

### 場景 2：內容創作加速器

當你要寫新文章時，可以問你的數字分身：
- 「根據我過去的觀點，這個新聞我應該怎麼看？」
- 「這個標題符合我一貫的風格嗎？」
- 「幫我檢查這篇草稿，有沒有違背我之前的立場？」

它不是代替你寫作，而是**確保你的輸出品質和風格一致性**。

### 場景 3：決策顧問

當你面臨選擇時：
- 「我想推出這個新產品，根據我過去的判斷標準，這是個好主意嗎？」
- 「這個合作邀請，我應該接受嗎？」

你的數字分身會基於你過去的言行，給出「另一個你」的觀點。

## NotebookLM：被低估的 RAG 革命

這個案例最讓我興奮的不是「數字分身」本身，而是 **NotebookLM 展示的 RAG 新範式**。

Google 用 NotebookLM 證明了一件事：**RAG 不需要這麼複雜。**

| 傳統 RAG | NotebookLM |
|---------|-----------|
| 要花 3 天搭建 | 3 分鐘上傳文件 |
| 要懂向量資料庫 | 會打字就能用 |
| 要維護 Embedding | Google 自動處理 |
| 要寫檢索語法 | 自然語言對話 |
| 更新資料要重新索引 | 添加文件即可 |

這對內容創作者是巨大的解放。**我們可以把精力從「維護技術架構」轉移到「創造優質內容」。**

當然，NotebookLM 也有限制：
- 資料要上傳到 Google（隱私考量）
- 沒有 API（無法自動化）
- 免費版有容量上限

但對於個人知識管理和內容創作，它已經足夠強大。

## 結語：未來已來，只是尚未均勻分佈

MinLi 的這個實作給了我們一個重要的啟示：**打造 AI 分身不再需要深厚的技術背景。**

過去你要：
- 會寫程式抓取資料
- 懂向量資料庫
- 會調 Prompt
- 有 GPU 或 API 額度

現在你要：
- 會拖放文件
- 會打字聊天
- 會觀察和提取「風格」

技術門檻的降低，意味著**「數字分身」這件事將從極客玩具變成大眾工具。**

想象一下：
- 每個知識工作者都有一個 AI 分身協助創作
- 每個專家都可以 24 小時「線上」回答問題
- 每個人的知識和經驗都可以被「複製」和「傳承」

這是機會，也是挑戰。

機會在於，你的價值不再受限於你的時間；  
挑戰在於，**當複製變得容易，原創的價值將更加珍貴。**

你準備好了嗎？
